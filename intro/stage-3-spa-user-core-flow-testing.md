# 阶段 3 SPA 用户核心闭环测试说明

## 本阶段变更概览

阶段 3 的目标是把用户端最关键的交易闭环迁移到真正可用的 React SPA 页面，并补齐列表筛选、分页、详情动作和提交后的局部刷新反馈。当前阶段已经把“登录后浏览信息流 -> 浏览任务 -> 发布需求 -> 接单 -> 确认完成 -> 评价”这条主链路做成可在 SPA 内完成的连续流程。

本阶段完成的关键变化：

- `frontend/src/spa/pages/FeedPage.tsx` 升级为可查询的信息流页面，支持关键词、状态、分类筛选和分页。
- `frontend/src/spa/pages/RequestsPages.tsx` 重做了任务列表、发布页和需求详情页。
- 发布页新增分类、标签、预计耗时、时间币、地点和文件上传等字段，并保留 AI 草稿生成和发布成功提示。
- 需求详情页支持接单、评论、发布者资料跳转和接单后进入订单页。
- `frontend/src/spa/pages/OrdersPages.tsx` 重做了订单列表、订单详情和评价页。
- 订单详情页支持确认完成、查看双方状态、进入评价与纠纷入口。
- `frontend/src/spa/pages/shared.tsx` 新增分页控件和若干通用显示工具，统一列表页的 query 处理和状态展示。
- `frontend/src/spa/styles.css` 补充了本阶段页面需要的表单、详情和分页样式。
- `scripts/validate-stage-03.mjs` 改成阶段 3 专用校验，除了静态代码约束，还会跑完整的核心 API 闭环。
- 新增组件测试和 e2e 测试，覆盖发布、接单、确认和评价的完整流程。

## 自动化验证

在项目根目录执行：

```powershell
npm run typecheck
npm run build
npm run test:component
npm run test:stage03
npm run test:e2e
npm run test:frontend-build
```

预期结果：

- 所有命令退出码为 0。
- `test:stage03` 应确认用户端核心页面存在、列表页支持筛选和分页、发布页能读取分类和标签、详情页支持接单和确认、评价页能读取 `orderId`。
- `test:e2e` 应覆盖生产运行时流程和 SPA 用户闭环流程。

## 手工验证环境

先构建前端产物：

```powershell
npm run build
```

启动 SPA 模式本地服务：

```powershell
$env:FRONTEND_MODE="spa"
npm run dev
```

默认访问地址：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:3001`

## 手工验收清单

### 1. 登录后进入首页信息流

访问 `/login`，使用测试账号登录后进入 `/feed`。

预期结果：

- 页面加载信息流数据。
- 可以通过搜索、状态和分类筛选列表。
- 翻页只更新当前列表，不发生整页刷新。

### 2. 发布需求

进入 `/post`，填写标题、描述、类别、预计耗时、时间币、地点和标签后提交。

预期结果：

- 表单提交成功。
- 页面显示发布成功提示。
- 可以直接跳转到新发布需求的详情页或继续留在 SPA 内操作。

### 3. 任务列表与接单

使用另一账号进入 `/tasks`，搜索刚发布的需求并打开详情页。

预期结果：

- 详情页展示需求信息、分类、发布者和评论区。
- 点击“接单”后进入对应订单页。
- 不出现整页跳转或浏览器刷新感。

### 4. 订单确认

在订单详情页分别用双方账号完成确认。

预期结果：

- 订单状态在当前页面局部更新。
- 页面保留在订单详情路由。
- 确认完成后能看到评价入口或当前可操作状态。

### 5. 评价提交

通过评价入口进入 `/reviews/new?orderId=...`，提交评分、标签和评论。

预期结果：

- 评价提交成功。
- 页面显示“评价已提交”。
- 评价后仍在 SPA 内完成后续导航。

### 6. 刷新与直达

直接刷新以下地址：

- `/feed`
- `/tasks`
- `/post`
- `/posts/:id`
- `/orders/:id`
- `/reviews/new?orderId=...`

预期结果：

- 页面可直接恢复到对应 React 视图。
- 不应回退到原型 HTML 页面。

## 仍不属于本阶段的范围

以下内容不是本阶段的核心验收点，会在后续阶段继续完善：

- 消息、通知、钱包、纠纷、陪审。
- 管理后台的完整生产等价迁移。
- 更细的视觉统一和复杂表单的二次优化。

## 测试结论记录建议

测试人员记录问题时建议包含：

- 当前账号。
- 当前路由和操作步骤。
- 是否发生整页刷新。
- 是否能够从刷新后直接回到同一 React 页面。
- Network 面板中是否出现新的 `document` 请求。
- 接单、确认、评价后的状态是否只做局部更新。
